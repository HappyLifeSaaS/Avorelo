# Governance

Avorelo is an Open Source project under the Apache License 2.0.

## Roles

- **Maintainers** review and merge contributions, set direction, and cut releases. See
  [MAINTAINERS.md](MAINTAINERS.md).
- **Contributors** submit issues and pull requests. Anyone can contribute (see
  [CONTRIBUTING.md](CONTRIBUTING.md)).

## Decision-making

The project is currently maintained by a single maintainer. Decisions are made pragmatically, favoring
the project's principles: local-first, honest about what it does, no hidden cost, and a small, focused
surface. Substantial changes should start as an issue so the approach can be discussed before work
begins.

## Merging

Every pull request requires: passing CI (build, tests, dogfood, site checks, licensing and contribution
truth gates, and DCO sign-off), resolved review conversations, and maintainer approval. Force-pushes to
and deletion of the default branch are disabled. Maintainers may use an administrative bypass only for
emergencies, and such use is auditable.

## Adding maintainers

A sustained, high-quality contributor may be invited to become a maintainer at the discretion of the
existing maintainer(s). If more than one maintainer is active, pull requests require at least one
approving review from someone other than the author.

## License and contributions

The software is licensed under Apache-2.0. Ordinary contributions are accepted under Apache-2.0 with a
DCO sign-off and do **not** require a CLA. An exceptional contribution may require a separate agreement,
as described in [CONTRIBUTING.md](CONTRIBUTING.md).

## Code of conduct

All participation is governed by the [Code of Conduct](CODE_OF_CONDUCT.md).
